# Stage-B GNN+RL Training

This directory contains the current Stage-B offline BC / offline PPO / online PPO training code.

Canonical project docs:

- `Doc/current_stageB_effective_configuration.md`
- `Doc/stageB_gnn_rl_online_training_design.md`
- `Doc/stageB_gnn_rl_training_implementation_tasklist.md`

## 1. Prerequisites

- Python 3.8+
- PyTorch (`torch`)
- NumPy (`numpy`) for online bridge client/trainer
- Matplotlib (`matplotlib`) for training curve plotting (optional but recommended)
- Replay dump generated by Stage-B M0 (`rl_replay_meta.json`, `rl_replay_schema.json`, `rl_replay_records.bin`)

## 2. Replay sanity check

```bash
python3 cuMAC/scripts/inspect_rl_replay.py --replay-dir output/replay_m0/RAYLEIGH
```

## 3. Behavior Cloning training

Single replay shard:

```bash
python3 training/gnnrl/bc_train.py \
  --replay-dir output/replay_m0/RAYLEIGH \
  --out-dir training/gnnrl/checkpoints/m1_bc_seed42 \
  --epochs 30 \
  --batch-size 64 \
  --lr 3e-4 \
  --seed 42
```

Multiple replay shards (same schema/dims):

```bash
python3 training/gnnrl/bc_train.py \
  --replay-dir output/replay_seed42/RAYLEIGH output/replay_seed43/RAYLEIGH \
  --out-dir training/gnnrl/checkpoints/m1_bc_multi \
  --epochs 40 \
  --batch-size 64
```

Outputs:

- `checkpoint_best.pt`
- `checkpoint_last.pt`
- `train_summary.json`

## 4. Export ONNX

```bash
python3 training/gnnrl/export_onnx.py \
  --checkpoint training/gnnrl/checkpoints/m1_bc_seed42/checkpoint_best.pt \
  --out training/gnnrl/checkpoints/m1_bc_seed42/model.onnx
```

## 5. Current scope / limitations

- Current BC pipeline supports replay `alloc_type=0` only.
- Current BC pipeline expects `n_tot_cell == n_cell`.
- Replay v2 adds `action_mask_cell_ue` (`[n_cell, n_active_ue]`) for true cell-UE legality.
- Legacy replay (v1) is still supported with heuristic fallback mask reconstruction.
- Target legality is enforced by masks; illegal targets are ignored during loss.
- This is offline BC (M1), not online RL training (M2).

## 6. Masked PPO (M2 MVP)

Warm-start from M1 BC checkpoint:

```bash
python3 training/gnnrl/ppo_train.py \
  --replay-dir output/replay_m0/RAYLEIGH \
  --init-policy-checkpoint training/gnnrl/checkpoints/m1_bc_seed42/checkpoint_best.pt \
  --out-dir training/gnnrl/checkpoints/m2_ppo_seed42 \
  --iters 50 \
  --ppo-epochs 6 \
  --minibatch-size 128 \
  --seed 42
```

Outputs:

- `ppo_actor_best.pt` (can be exported to ONNX with `export_onnx.py`)
- `ppo_actor_last.pt`
- `ppo_train_last.pt`
- `ppo_summary.json`

Example export after M2:

```bash
python3 training/gnnrl/export_onnx.py \
  --checkpoint training/gnnrl/checkpoints/m2_ppo_seed42/ppo_actor_best.pt \
  --out training/gnnrl/checkpoints/m2_ppo_seed42/model.onnx \
  --opset 18
```

## 7. Online PPO bridge (M3 bootstrap)

The online bridge protocol is defined in:

- `cuMAC/examples/onlineTrainBridge/OnlineProtocol.h`
- `training/gnnrl/online_protocol.py`

Client and trainer:

- `training/gnnrl/aerial_env.py`
- `training/gnnrl/ppo_online_train.py`

Example:

```bash
python3 training/gnnrl/ppo_online_train.py \
  --sim-bin build.x86_64/cuMAC/examples/multiCellSchedulerUeSelection/multiCellSchedulerUeSelection \
  --sim-args "-d 1 -b 0 -x 0 -f 0 -g 100 -r 5000" \
  --init-policy-checkpoint training/gnnrl/checkpoints/m1_bc_seed42/checkpoint_best.pt \
  --online-persistent 1 \
  --plot-after-train 1 \
  --plot-smooth-window 5 \
  --out-dir training/gnnrl/checkpoints/m3_online_ppo_seed42 \
  --episode-horizon 400 \
  --rollout-steps 256 \
  --iters 20
```

Notes:

- `--online-persistent 1` (default): keep one simulator subprocess alive and stream continuous TTIs.
- `--episode-horizon` only affects `--online-persistent 0`.
- Training outputs also include:
- `online_ppo_metrics.csv` (per-iter metrics table)
- `online_ppo_curves.png` (loss/kl/entropy/reward/clipfrac curves; requires matplotlib)
